# 文件命名规范

为方便查找、版本管理和团队协作，建议所有项目文件按以下格式命名：

```text
[项目缩写]_[阶段]_[版本]_[文件说明]
```

## 命名示例

```text
P001_RD_V2_需求文档.md
```

## 字段说明

- `项目缩写`：项目编号或简称，例如 `P001`。
- `阶段`：项目阶段，例如 `RD` 表示需求阶段，`DES` 表示设计阶段，`DEV` 表示开发阶段。
- `版本`：文件版本，例如 `V1`、`V2`、`V3`。
- `文件说明`：简要说明文件内容，例如 `需求文档`、`设计说明`、`测试记录`。

## 建议

- 文件名尽量简短清晰。
- 同一项目保持统一项目缩写。
- 修改重要文件时更新版本号。
